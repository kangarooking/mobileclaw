========================================
 MobileClaw 项目进度日志
 基于长时运行 Agent 方法论
========================================

项目概述:
  iOS/Android 多模态语音+视觉 OpenClaw 移动端客户端。
  语音唤醒 → 相机+麦克风同时激活 → 实时画面+语音发给 openclaw → AI 回复 + TTS 播放。
  目标用户：袋鼠帝（公众号写作者），日常需要高频与 AI Agent 交互。

技术栈:
  React Native + Expo Dev Client (Bare Workflow) + TypeScript
  Zustand 状态管理 / react-native-vision-camera v4 / React Navigation v6
  ASR: 豆包流式 API (主) / TTS: Edge TTS + openclaw tts.convert (双路径)
  后端协议: OpenClaw Gateway WebSocket (JSON req/res/event, protocol v3)

配色/设计方向:
  暗色主题 (#0a0a0f 背景)，工具优先极简 UI
  相机预览占屏幕 ~55%，聊天记录在下方
  录制状态红色脉冲边框，TTS 播放紫色波形动画

协议发现（关键！影响后续所有功能）：
  - client.id 必须是预定义枚举值（12个），iOS 客户端用 'openclaw-ios'（不能自定义如 'mobileclaw-ios'）
  - client.mode 枚举值：'webchat'|'cli'|'ui'|'backend'|'node'|'probe'|'test'
    MobileClaw 作为 UI 客户端应使用 'ui'
  - 设备身份（Device Identity）是获取 operator.write 权限的**必要条件**
    无 device → scopes 被清空为 [] → 所有 RPC 返回 "missing scope"
  - 设备身份使用 Ed25519 签名算法（非 ECDSA P-256！）
  - v3 签名 payload 格式：v3|deviceId|clientId|clientMode|role|scopes|signedAtMs|token|nonce|platform_norm|deviceFamily_norm
  - **关键坑**：connectParams.client 中的字段必须与签名 payload 中使用的完全一致
    否则服务端重建的 payload 与客户端签名的不匹配 → "device signature invalid"
    特别是 deviceFamily：如果在签名中用了 'iPhone'，connectParams.client 也必须有 deviceFamily:'iPhone'
  - AI 对话方法为 chat.send（sessionKey+message+idempotencyKey）
    send 方法是给 WhatsApp/SMS 等频道消息用的（需要 to+channel 参数）
  - sessionKey 格式：'agentId:channel:peer'（至少3段冒号分隔），如 'main:webchat:mobileclaw'
  - 首次连接新设备自动静默配对（local client + shared auth → silent auto-approve）
  - 已配对设备请求更多 scope 时需重新配对

测试结果（11/11 通过）：
  ✅ TCP 连接到 ws://127.0.0.1:18789
  ✅ 收到 connect.challenge 事件（含 nonce）
  ✅ 发送 connect 请求（含 Ed25519 设备身份 + v3 签名 payload）
  ✅ 收到 hello-ok 响应（protocol:3, server version, features list）
  ✅ 获取 deviceToken + operator.read + operator.write 权限
